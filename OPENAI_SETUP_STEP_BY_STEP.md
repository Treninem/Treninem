# OpenAI API — пошаговое подключение

Ниже простой порядок действий, чтобы бот реально заработал.

## Шаг 1. Войти в OpenAI Platform

Открой `platform.openai.com` и войди в свой аккаунт.

## Шаг 2. Создать проект

1. Открой раздел `Projects`.
2. Нажми `New project`.
3. Назови проект, например: `maski-v1`.
4. Открой этот проект.

## Шаг 3. Подключить оплату

1. Открой `Billing`.
2. Добавь карту или другой доступный способ оплаты.
3. Если используешь prepaid billing, пополни баланс.
4. После пополнения подожди несколько минут, пока кредиты появятся.

## Шаг 4. Настроить лимиты проекта

1. Внутри проекта открой `Limits`.
2. Поставь удобный месячный бюджет и уведомления.
3. При необходимости включи нужные модели проекта.

## Шаг 5. Создать API key

1. Открой `API keys` внутри нужного проекта.
2. Нажми `Create new secret key`.
3. Скопируй ключ сразу — второй раз OpenAI его полностью не покажет.
4. Вставь этот ключ в `.env` в поле `OPENAI_API_KEY`.

## Шаг 6. Вписать ключ в `.env`

Пример:

```env
OPENAI_API_KEY=sk-proj-xxxxxxxxxxxxxxxxxxxxxxxx
OPENAI_TEXT_MODEL=gpt-4.1-mini
OPENAI_IMAGE_MODEL=gpt-image-1.5
OPENAI_VISION_MODEL=gpt-4.1-mini
```

## Шаг 7. Не допускать утечки ключа

Нельзя:
- отправлять ключ в Telegram;
- хранить ключ в публичном GitHub;
- давать ключ дочерним владельцам;
- вставлять ключ прямо в код.

Ключ должен лежать только в `.env` на сервере.

## Шаг 8. Проверить запуск

После заполнения `.env`:

```bash
pip install -r requirements.txt
python smoke_test.py
python -u bot.py
```

## Шаг 9. Проверить в Telegram

1. Открой root-бота.
2. Нажми `/start`.
3. Проверь текстовую генерацию.
4. Отправь фото и проверь маску.
5. Если всё работает — проект подключён правильно.

## Шаг 10. Если появляется ошибка биллинга

Если бот пишет, что достигнут лимит или закончился биллинг:
- зайди снова в `Billing`;
- пополни баланс или подними usage limit;
- подожди несколько минут;
- попробуй ещё раз.

## Шаг 11. Если ключ случайно утёк

1. Зайди в `API keys`.
2. Удали или отключи старый ключ.
3. Создай новый.
4. Замени `OPENAI_API_KEY` в `.env`.
5. Перезапусти бота.


Поддержка в боте:
- `OWNER_CONTACT_USERNAME` — username владельца, показывается пользователям в важных сообщениях.
- username администратора поддержки можно добавить позже прямо из root-админки: `🛡 Админ-сеть` → `👤 Контакты поддержки`.
