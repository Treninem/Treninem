# maski v1

Готовая версия Telegram-платформы на OpenAI API для Bothost.

## Что внутри

### Для пользователей
- 30 шаблонов масок и стилизаций.
- Маска по своему описанию.
- Подбор подходящих масок под фото.
- Тексты: стихи, песни, поэмы, поздравления, подписи, статусы, тосты.
- 5 страниц примеров масок.
- Платные пакеты через Telegram Stars.

### Для владельца платформы
- Админ-панель есть только у корневого владельца.
- Общая статистика по сети.
- Популярные маски и популярные запросы.
- Все пользователи списком, ссылки на профили, бан/разбан.
- Подозрительные пользователи и ручная проверка.
- Просмотр корневого бота и всех подключённых дочерних ботов.
- Выдача глобальных бонусных запросов вручную.

### Для дочерних владельцев
- Подключение своего бота по токену от BotFather.
- 1 бесплатный запрос в день только владельцу его дочернего бота.
- Небольшой реферальный бонус, если новый владелец пришёл по его ссылке.
- Нет доступа к сетевой админ-панели.

## Как устроена сеть

- Есть один корневой бот платформы.
- Все новые боты подключаются напрямую к корневому боту.
- Если новый владелец пришёл по ссылке от дочернего бота, этот дочерний владелец получает только небольшой реферальный бонус.
- Все основные продажи и вся платёжная логика всё равно учитываются у корневой платформы.
- Глубокой многоуровневой сети нет: схема сделана проще и рентабельнее.

## Комиссии по умолчанию

- владелец продающего дочернего бота: `12%`
- корневая линия: `5%`
- реферальный бонус пригласившему дочернему владельцу: `3%`
- платформа: `8%`

Значения можно менять в `.env`.

## Бесплатные лимиты

- новый обычный пользователь: `1` бесплатный запрос на каждом боте;
- владелец дочернего бота: `1` бесплатный запрос в день на своём боте;
- корневой админ: безлимит без списаний.

## Файлы проекта

- `bot.py` — основной runtime root-бота и дочерних ботов.
- `db.py` — база SQLite и бизнес-логика платформы.
- `openai_service.py` — работа с OpenAI API.
- `presets.py` — 30 шаблонов.
- `page_1.jpg` … `page_5.jpg` — 5 страниц карточек-примеров.
- `example_01.jpg` … `example_30.jpg` — отдельные демонстрационные примеры шаблонов.
- `PREVIEW_SOURCES.md` — список источников превью №2–№30.
- `OPENAI_SETUP_STEP_BY_STEP.md` — пошаговое подключение OpenAI API.
- `README_BOTHOST.md` — быстрый запуск на Bothost.

## Что нужно заполнить

Открой `.env` и укажи:
- `TELEGRAM_BOT_TOKEN`
- `OPENAI_API_KEY`
- `ADMIN_USER_IDS`
- `ROOT_OWNER_USER_ID`

## Локальный запуск

```bash
python -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
python bot.py
```

## Самопроверка

```bash
python smoke_test.py
```

Проверка проходит офлайн и проверяет:
- импорт проекта;
- создание базы;
- создание root и дочернего бота;
- квоты, платежи и статистику;
- наличие страниц превью.

## Важно

- Не запускай один и тот же Telegram-токен сразу в двух местах.
- Не передавай никому свой OpenAI API key.
- Если OpenAI вернёт ошибку биллинга, бот не должен списывать запросы у пользователей.
- Превью в архиве — это демонстрационные карточки под шаблоны. Финальный результат в проде зависит от исходного фото, выбранного шаблона и ответа OpenAI API.


Поддержка в боте:
- `OWNER_CONTACT_USERNAME` — username владельца, показывается пользователям в важных сообщениях.
- username администратора поддержки можно добавить позже прямо из root-админки: `🛡 Админ-сеть` → `👤 Контакты поддержки`.
