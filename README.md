# AI Mask Platform for Telegram

Многоуровневая платформа Telegram-ботов на базе OpenAI API и `python-telegram-bot`.

## Что умеет платформа

### Для пользователей
- 30 готовых трендовых масок.
- Маска по текстовому описанию пользователя.
- Подбор лучших масок под фото.
- Генерация стихов, песен, поэм, поздравлений, тостов, статусов и подписей.
- Примеры масок в виде preview-страниц.
- Telegram Stars для покупки пакетов и PRO.

### Для владельца платформы
- Только у root-владельца есть панель сети.
- Общая статистика по всей сети.
- Популярные запросы и популярные маски.
- Список всех пользователей со ссылками на профили.
- Глобальный бан / разбан.
- Глобальный VIP платформы.
- Выдача бонусных запросов вручную.
- Просмотр дерева дочерних ботов и карточек каждого бота.
- Уведомления о подозрительных пользователях.

### Для дочерних ботов
- Подключение бота по токену от BotFather.
- Автозапуск дочернего бота внутри общей платформы.
- У владельца дочернего бота есть 10 бесплатных запросов в день на своём боте.
- У обычных пользователей дочернего бота есть 5 бесплатных запросов на этом боте.
- Кнопка «Создать такого же бота» ведёт в root-бот и строит реферальное дерево.

## Важная архитектурная идея

Платежи проходят **через корневого бота платформы**, а не напрямую в дочерних ботах.

Почему это сделано так:
- у тебя остаётся единый платёжный центр;
- можно честно считать многоуровневые комиссии в одной базе;
- root-бот видит, какой дочерний бот привёл продажу;
- дочерние боты остаются лёгкими и безопасными.

То есть покупка инициируется из дочернего бота, но оплачивается через root-бота по deep-link.

## Комиссии

По умолчанию в `.env` включена такая схема:
- прямой владелец продающего дочернего бота: `20%`
- родительский уровень: `7%`
- третий уровень: `3%`
- корневая платформа: `10%`

Все начисления записываются в таблицу `commission_events`.

## Структура

- `bot.py` — единый runtime для root-бота и всех дочерних ботов.
- `app/db.py` — SQLite-модель платформы.
- `app/services/openai_service.py` — OpenAI image/text/vision.
- `assets/mask_previews/` — preview-страницы масок.

## Установка

```bash
python -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
cp .env.example .env
```

Заполни `.env`, затем запусти:

```bash
python bot.py
```

## Что обязательно заполнить в `.env`

- `TELEGRAM_BOT_TOKEN` — токен root-бота.
- `OPENAI_API_KEY` — ключ OpenAI.
- `ADMIN_USER_IDS` — твой Telegram user id.
- `ROOT_OWNER_USER_ID` — тоже твой Telegram user id.

## Как подключить дочерний бот

1. Открой root-бот.
2. Нажми **«Подключить своего бота»**.
3. Создай бота у `@BotFather`.
4. Вставь токен в мастер подключения.
5. Отправь красивое название для карточки в сети.
6. Платформа сама стартует нового бота без отдельного кода.

## Что важно знать

- Админ-панель есть только у root-владельца.
- Дочерние владельцы не получают сетевую админку.
- Если после добавления нового токена Telegram ограничит polling для токена, проверь, не запущен ли этот же токен где-то ещё.
- В одной системе не должен параллельно работать тот же токен в другом процессе.

## Быстрая самопроверка

После заполнения `.env` можно запустить локальную проверку без Telegram и без реальных запросов в OpenAI:

```bash
python smoke_test.py
```

Она проверит:
- импорт проекта и зависимостей;
- инициализацию SQLite;
- создание root и дочерних ботов в БД;
- квоты, платежи, комиссии и статистику;
- наличие preview-файлов масок.

## Рекомендации по продакшену

- Лучше запускать платформу на VPS или сервере с постоянным процессом.
- Для большой сети лучше перейти с polling на webhook.
- Делай регулярные бэкапы `data/bot.sqlite3`.
- Не хранить чужие токены в логах и не пересылать их никому.

## Проверено в этой версии

- синтаксис проекта через `python -m py_compile`
- единая SQLite-схема платформы
- логика root-бота / дочерних ботов / комиссий / deep-link платежей

## Ограничение текущей версии

Эта версия делает **управляемую white-label платформу на токенах BotFather**.
Поддержка Telegram Managed Bots как отдельного автоматического канала создания **не включена**, чтобы сборка оставалась стабильной и запускаемой на обычном сервере без дополнительной Telegram-специфики.
