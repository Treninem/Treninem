# Запуск на Bothost

## 1. Создай проект

В Bothost создай новый Python-проект и загрузи в него содержимое этой папки.

## 2. Проверь корень проекта

В корне проекта должны лежать:
- `bot.py`
- `requirements.txt`
- `.env`
- `Procfile`
- `runtime.txt`
- `start.sh`

## 3. Заполни `.env`

Минимально обязательно:

```env
TELEGRAM_BOT_TOKEN=токен_корневого_бота
OPENAI_API_KEY=твой_openai_api_key
ADMIN_USER_IDS=твой_telegram_id
ROOT_OWNER_USER_ID=твой_telegram_id
```

## 4. Установка зависимостей

Команда установки:

```bash
pip install -r requirements.txt
```

## 5. Команда запуска

```bash
python -u bot.py
```

## 6. Версия Python

Рекомендуется Python `3.11`.

## 7. После первого запуска

- база создастся автоматически в `data/bot.sqlite3`;
- root-бот появится в системе автоматически;
- далее можно подключать дочерние боты через меню root-бота.

## 8. Если `.env` не видно

Файл скрытый. Если панель хостинга его не показывает, используй:
- `env_for_bothost.txt`
- `ENV_VISIBLE.txt`

Скопируй их содержимое в `.env`.

## 9. Если бот не отвечает

Проверь:
- правильный ли `TELEGRAM_BOT_TOKEN`;
- правильный ли `OPENAI_API_KEY`;
- не запущен ли тот же Telegram-токен ещё где-то;
- установились ли зависимости без ошибок.

## 10. Если OpenAI пишет про биллинг

Значит проблема не в Telegram-боте, а в аккаунте OpenAI API. Тогда нужно пополнить биллинг или поднять лимиты в OpenAI Platform.


Поддержка в боте:
- `OWNER_CONTACT_USERNAME` — username владельца, показывается пользователям в важных сообщениях.
- username администратора поддержки можно добавить позже прямо из root-админки: `🛡 Админ-сеть` → `👤 Контакты поддержки`.
