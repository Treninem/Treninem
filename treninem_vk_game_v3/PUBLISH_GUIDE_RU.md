# Подробная публикация Beast Legends by Treninem

## 1. Что в архиве

- `backend/` — сервер игры на FastAPI + SQLite.
- `backend/static/` — фронтенд VK Mini App.
- `android_wrapper/` — Android Studio проект для Android-версии.
- `main.py` — локальный запуск сервера.
- `requirements.txt` — Python-зависимости.

---

## 2. Какие данные тебе понадобятся

### Для VK Mini App
Нужно получить:
- **ID приложения VK Mini App**
- **Защищённый ключ приложения VK**
- **свой VK ID**
- **публичный HTTPS URL**, на котором будет работать сервер

Эти данные вставляются в файл:

`backend/config.py`

Поля:

```python
VK_APP_ID = 1234567
VK_APP_SECRET = 'PASTE_VK_APP_SECRET_HERE'
PUBLIC_WEB_URL = 'https://example.com'
OWNER_VK_IDS = [123456789]
ADMIN_PASSWORD = 'CHANGE_ME_SUPER_SECRET'
```

### Для Android / Google Play
Нужно будет подготовить:
- Google аккаунт разработчика
- название приложения
- package name, например `com.treninem.beastlegends`
- иконку приложения 512x512
- feature graphic
- скриншоты
- политику конфиденциальности
- email поддержки
- URL backend

---

## 3. Где брать данные для VK

### 3.1. ID приложения VK Mini App
После создания приложения VK в кабинете разработчика ты получишь **App ID**.
Это число и есть значение для:

```python
VK_APP_ID = 1234567
```

### 3.2. Защищённый ключ VK приложения
В настройках созданного приложения VK есть **защищённый ключ**.
Он нужен для проверки подписи `sign` и серверной авторизации игрока.
Вставляй сюда:

```python
VK_APP_SECRET = 'ТВОЙ_ЗАЩИЩЕННЫЙ_КЛЮЧ'
```

### 3.3. OWNER_VK_IDS
Это твой личный VK ID.
Его можно узнать:
- из адреса своего профиля,
- через любое определение VK ID,
- через dev-инструменты VK.

Пример:

```python
OWNER_VK_IDS = [123456789]
```

### 3.4. PUBLIC_WEB_URL
Это публичный HTTPS-адрес, где реально открыт сервер игры.
Примеры:

```python
PUBLIC_WEB_URL = 'https://game.treninem.ru'
```
или
```python
PUBLIC_WEB_URL = 'https://beast-legends.onrender.com'
```

## 5. Как подготовить сервер к публикации VK-версии

## Вариант A — VPS / свой сервер
Подходит лучше всего.

Что нужно:
- Linux VPS
- Python 3.11+
- домен
- HTTPS через Nginx + SSL

Шаги:
1. Загрузи проект на сервер.
2. Создай виртуальное окружение.
3. Установи зависимости из `requirements.txt`.
4. В `backend/config.py` вставь реальные данные VK и свой домен.
5. Запусти через `uvicorn` или через `systemd`.
6. Настрой Nginx reverse proxy.
7. Подключи SSL сертификат.

Пример запуска:

```bash
uvicorn backend.app:app --host 0.0.0.0 --port 8000
```

## Вариант B — облачный хостинг
Можно использовать Render, Railway, Fly.io, свой Docker-хостинг или другой VPS.

Главное требование:
- приложение должно быть доступно по **HTTPS**
- URL должен совпадать с тем, что указан в `PUBLIC_WEB_URL`

## 6. Как подключить VK Mini App

### 6.1. Создай VK Mini App
После создания приложения в кабинете разработчика:
- получишь App ID
- получишь доступ к настройкам Mini App

### 6.2. Укажи URL приложения
В настройках VK Mini App укажи URL, который открывает фронтенд приложения.
В этом проекте фронтенд уже раздаётся через backend, поэтому можно использовать корневой URL:

```text
https://твойдомен.ru/
```

### 6.3. Проверь config.py
Убедись, что:
- `VK_APP_ID` верный
- `VK_APP_SECRET` верный
- `PUBLIC_WEB_URL` совпадает с реальным HTTPS URL

### 6.4. Проверка авторизации
При открытии приложения из VK в URL передаются `vk_*` параметры и `sign`.
Сервер уже умеет проверять подпись и создавать/восстанавливать профиль пользователя.

## 7. Как опубликовать приложение в VK

Обычно порядок такой:
1. Создать приложение.
2. Заполнить карточку приложения.
3. Указать название, описание, категорию.
4. Загрузить иконки и обложки.
5. Подключить рабочий HTTPS URL.
6. Проверить авторизацию, загрузку и стабильность.
7. Отправить приложение на модерацию.

Подготовь заранее:
- название: **Beast Legends**
- создатель: **Treninem**
- короткое описание
- полное описание
- иконку
- баннер/обложку
- контакты поддержки
- политику конфиденциальности

---

## 8. Как работает синхронизация VK ↔ Android

### 8.1. В VK-версии
Игрок нажимает кнопку:
- **Код для Android**

Сервер создаёт одноразовый код.

### 8.2. В Android-приложении
Пользователь:
- вводит URL backend
- вставляет одноразовый код

После этого Android получает тот же профиль.

---

## 9. Как собрать Android-версию

Открой папку:

`android_wrapper/`

в Android Studio.

### 9.1. Перед сборкой
Проверь:
- package name
- название приложения
- иконки
- backend URL по умолчанию

Файл:

`android_wrapper/app/src/main/java/com/treninem/beastlegends/MainActivity.kt`

Строка:

```kotlin
backendUrlInput.setText(prefs.getString("backend_url", "https://example.com"))
```

Поставь сюда свой URL.

### 9.2. Сборка APK / AAB
В Android Studio:
- **Build > Generate Signed Bundle / APK**
- выбери **Android App Bundle (AAB)** для Google Play
- создай или укажи keystore
- собери релиз

## 10. Как опубликовать в Google Play

Нужно:
1. создать аккаунт Google Play Console;
2. заполнить данные разработчика;
3. пройти верификацию;
4. создать приложение;
5. загрузить AAB;
6. заполнить карточку магазина;
7. указать privacy policy;
8. заполнить Data safety;
9. пройти тестирование;
10. отправить релиз на проверку.

---

## 11. Где брать данные для Google Play

### 11.1. Название приложения
Придумываешь сам.
Рекомендую:
- `Beast Legends`

### 11.2. Package name
Это внутренний идентификатор Android-приложения.
Например:
- `com.treninem.beastlegends`

Его лучше не менять после начала публикации.

### 11.3. Иконка приложения
Нужно подготовить:
- launcher icon
- store icon 512x512

### 11.4. Скриншоты
Сделай реальные скриншоты:
- главного экрана
- профиля
- инвентаря
- рынка
- крафта
- админки не нужно

### 11.5. Privacy Policy
Нужна отдельная страница с политикой конфиденциальности.
Даже если ты хранишь только игровой профиль, лучше сделать простую страницу с описанием:
- какие данные хранятся;
- зачем;
- как удалить аккаунт;
- куда писать по вопросам.

### 11.6. Support email
Нужен email поддержки, например:
- `support@treninem.ru`

## 12. Как менять игру после публикации

### Для VK
- обновляешь backend
- проверяешь, что URL не изменился
- при необходимости обновляешь карточку приложения

### Для Android
- меняешь код
- собираешь новую версию AAB
- загружаешь новый релиз в Google Play Console

## 13. Пароль админки

В файле `backend/config.py`:

```python
ADMIN_PASSWORD = 'CHANGE_ME_SUPER_SECRET'
```

После запуска:
- зайди своим аккаунтом владельца
- открой экран **Админка**
- введи пароль

## 14. Что обязательно сделать перед публикацией

- заменить все тестовые значения в `backend/config.py`
- выключить `ALLOW_DEV_LOGIN = False`
- поставить сильный пароль админки
- поднять backend на HTTPS
- проверить вход через реальный VK Mini App
- проверить подарки, рынок, крафт, банк, PvP
- проверить Android-код синхронизации
- сделать резервную копию `game.db`
